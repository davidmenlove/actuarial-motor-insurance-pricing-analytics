# Results

This directory contains the curated analytical outputs generated by the **Actuarial Motor Insurance Pricing Analytics** project.

Whereas the `outputs/` directory stores intermediate working files created during execution, the `results/` directory contains organized figures, tables, and model artifacts intended for documentation, reporting, and portfolio presentation.

---

# Directory Structure

```text
results/
│
├── figures/
├── tables/
├── model_artifacts/
├── results_catalog.md
└── README.md
```

---

# Contents

## Figures

The `figures/` directory contains visualizations generated during the analytical workflow.

Typical outputs include:

- Claim severity distribution comparisons
- Monte Carlo simulation results
- Pricing scenario visualizations
- Statistical model diagnostics

---

## Tables

The `tables/` directory contains summary statistics and analytical outputs used throughout the project.

Examples include:

- Pricing summaries
- Economic scenario comparisons
- Frequency and severity statistics
- Value-at-Risk (VaR) and Conditional Value-at-Risk (CVaR) summaries

---

## Model Artifacts

The `model_artifacts/` directory contains selected outputs from the statistical modeling process that are preserved for documentation and reproducibility.

Examples include:

- Selected severity model
- Candidate model comparison
- Distribution parameter estimates

---

## Results Catalog

The `results_catalog.md` document serves as an index of all generated analytical outputs.

Each entry briefly describes the purpose of the corresponding figure or table and how it relates to the overall pricing workflow.

---

# Purpose

The project intentionally separates generated outputs into two locations.

## outputs/

Working directory used by the automated analytics pipeline.

Contents are regenerated whenever the project is rebuilt.

---

## results/

Curated collection of presentation-ready outputs intended for:

- Technical documentation
- Project reporting
- Portfolio presentation
- GitHub reference material

This separation allows the analytical pipeline to remain reproducible while maintaining a clean and organized collection of final deliverables.

---

# Reproducing the Results

Run the complete project pipeline:

```bash
python build_project.py
```

The build process automatically:

1. Creates the DuckDB data warehouse
2. Executes the actuarial pricing models
3. Runs the Monte Carlo simulation
4. Generates analytical figures and tables
5. Updates the Power BI datasets
6. Refreshes the curated results contained in this directory
