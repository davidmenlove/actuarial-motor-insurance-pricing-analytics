# Power BI Dashboard Guide

This folder contains the Power BI report, dashboard screenshots, and the datasets automatically generated by the actuarial analytics pipeline.

The dashboards present the analytical results in a format intended for business stakeholders, demonstrating how statistical modeling can be translated into actionable pricing insights.

# Dashboard Overview

The report currently contains two dashboard pages.

## 1. Executive Pricing Dashboard

This page summarizes the key outputs of the actuarial pricing model.

### Primary Objectives

- Present the portfolio overview
- Estimate annual pure premium
- Compare economic pricing scenarios
- Quantify annual loss uncertainty
- Communicate pricing recommendations

### Key Performance Indicators

- Policies
- Exposure Years
- Claim Frequency
- Pure Premium
- Selected Severity Model
- Base Scenario Premium
- Stress Scenario Premium
- Premium Change

### Visualizations

- Pure Premium by Economic Scenario
- Annual Loss Value-at-Risk (99.5%)
- Annual Loss Conditional Value-at-Risk (99.5%)
- Executive Summary

---

## 2. Statistical Model Validation

This page documents the statistical modeling process used to select the severity distribution.

### Primary Objectives

- Compare candidate severity distributions
- Validate model selection using AIC and BIC
- Display fitted distribution parameters
- Compare observed and fitted claim distributions

### Visualizations

- Candidate Severity Models
- Selected Severity Model
- Parameter Estimates
- Observed vs. Fitted Severity Distribution
- Model Validation Summary

# Data Pipeline

The Power BI report is automatically populated from datasets generated by the project build pipeline.

```text
DuckDB Warehouse
        │
        ▼
Python Analytics
        │
        ▼
Power BI Dataset Export
        │
        ▼
powerbi/datasets/
        │
        ▼
Power BI Report
```

Running

```bash
python build_project.py
```

automatically refreshes every dataset required by the report.

# Dashboard Design Philosophy

The dashboards were designed for two audiences.

### Executive Dashboard

Focuses on high-level pricing decisions.

Questions answered:

- What is the expected annual claims cost?
- How sensitive is pricing to inflation?
- How much tail risk exists?
- Which pricing scenario should be considered?

---

### Statistical Validation Dashboard

Focuses on analytical transparency.

Questions answered:

- Why was the Lognormal distribution selected?
- How well do candidate models perform?
- What parameters define the fitted distributions?
- How closely does the selected model match observed claims?
